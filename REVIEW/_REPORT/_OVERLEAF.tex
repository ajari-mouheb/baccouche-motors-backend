\section{Architecture}
\label{sec:architecture}

\subsection{Overall System Architecture}

The application follows a microservices architecture with an API Gateway pattern.

\bigskip
\noindent\textbf{Diagram Description:}
\begin{itemize}
    \item \textbf{Frontend Layer}: Web Browser interacts with Next.js application
    \item \textbf{API Gateway}: NestJS Gateway on port 3500 handles routing and authentication
    \item \textbf{Message Broker}: RabbitMQ on port 5672 routes requests between services
    \item \textbf{Microservices}: Auth, Cars, News, Test Drives, Contacts, and Admin services
    \item \textbf{Database Layer}: PostgreSQL with polyglot persistence (separate databases per service)
\end{itemize}

\begin{figure}[h]
\centering
\fbox{\parbox{0.9\textwidth}{
\textbf{High-Level Architecture Diagram}\\
\begin{center}
Web Browser $\rightarrow$ API Gateway (:3500) $\rightarrow$ RabbitMQ (:5672) $\rightarrow$ Microservices \\
$\rightarrow$ PostgreSQL Databases
\end{center}
}}
\end{figure}

\subsection{Backend Microservices Architecture}

\bigskip
\noindent\textbf{Components:}
\begin{itemize}
    \item \textbf{Gateway}: Port 3500 - Entry point with rate limiting and auth
    \item \textbf{Auth Service}: Port 3501 - User authentication
    \item \textbf{Cars Service}: Port 3502 - Car inventory management
    \item \textbf{News Service}: Port 3503 - News/Blog management
    \item \textbf{Test Drives Service}: Port 3504 - Test drive scheduling
    \item \textbf{Contacts Service}: Port 3505 - Contact form submissions
    \item \textbf{Admin Service}: Port 3506 - Admin dashboard
\end{itemize}

\subsection{Frontend Architecture}

\bigskip
\noindent\textbf{Route Structure:}
\begin{itemize}
    \item \textbf{Public Routes}: /, /about, /cars, /cars/[slug], /contact, /test-drive, /services, /actualites
    \item \textbf{Admin Routes (Protected)}: /admin/login, /admin/dashboard, /admin/cars, /admin/news, /admin/contacts, /admin/test-drives
    \item \textbf{Customer Routes (Protected)}: /customer/login, /customer/register, /customer/dashboard, /customer/profile, /customer/test-drives
\end{itemize}

\subsection{Data Flow Architecture}

\bigskip
\noindent\textbf{Flow Description:}
\begin{enumerate}
    \item User navigates to route
    \item Page renders component
    \item Component calls React Query hook
    \item Hook makes HTTP Request via Axios
    \item API Gateway processes request
    \item Response updates cache
    \item Component re-renders with data
    \item User sees updated UI
\end{enumerate}

\subsection{Authentication Flow}

\bigskip
\noindent\textbf{Auth Sequence:}
\begin{enumerate}
    \item User loads app
    \item AuthContext checks for token in sessionStorage
    \item If token exists: validate via /api/auth/me
    \item If valid: show protected content
    \item If invalid: show public content
    \item On login: POST credentials $\rightarrow$ receive JWT
    \item Store in sessionStorage and redirect
\end{enumerate}

\subsection{Sequence Diagrams}

\bigskip
\noindent\textbf{Login Sequence:}
\begin{enumerate}
    \item User enters credentials
    \item Frontend calls AuthContext.login()
    \item API makes POST to /api/auth/login
    \item Gateway forwards to Auth Service (RabbitMQ)
    \item Auth Service validates against database
    \item Returns JWT token
    \item Stored in sessionStorage
    \item Redirect to dashboard
\end{enumerate}

\bigskip
\noindent\textbf{Book Test Drive Sequence:}
\begin{enumerate}
    \item Customer fills form
    \item Submits to /api/test-drives
    \item Gateway routes to Test Drive Service
    \item Saves to database
    \item Sends confirmation
    \item Returns success to frontend
\end{enumerate}
